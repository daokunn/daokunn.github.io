MySQL的安装启动实践

关闭防火墙、关闭selinux

确保没有mysql其他版本

一、检查

Yum检查
yum list installed | grep mysql

安装则直接删除

yum remove mysql mysql-server mysql-libs compat-mysql

yum remove mysql-community-release

rpm检查
rpm -qa | grep -i mysql

有则直接删除

rpm -e --nodeps mysql-community-libs-5.7.22-1.el7.x86_64

rpm -e –nodeps mysql57-community-release-el7-11.noarch

二、口令查找Mysql的安装目录和残存文件

  1. whereis mysql

  2. find / -name mysql

找到后，全部rm删除。

MySQL8的配置及数据初始化

一般nginx,Redis这些，安装完，配置一下启动就好。但是Mysql还有数据初始化才可以启动。也就是MySQL需要:

  • 配置
  • 数据初始化

配置

vim /etc/my.cnf

数据初始化

  1. 新增用户

    useradd mysql -s /sbin/nologin

  2. 数据目录

    mkdir -pv /data/mysql

    chown -R mysql:mysql /data/mysql /usr/local/mysql/

  3. 初始化

    mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/data/mysql

    我的报错了，如下(这是两个报错)

    解决办法，缺少装啥

    yum install -y libaio.so.1

    yum install libnuma.so.1

    成功了

  4. 需要记住临时密码：A temporary password is generated for root@localhost: jR=cxEs3LLRl

  5. 验证是否初始化成功：/data/mysql

MySQL8服务器启动

  1. 脚本启动：/etc/init.d/mysqld start

  2. 脚本关闭：/etc/init.d/mysqld stop

  3. 手工启动：nohup mysqld_safe –defaultsfile=/etc/my.cnf &

  4. 手工关闭：kill

解决:cp /usr/local/mysql/.support-files/mysql.server /etc/init.d/mysqld

  2. 启动：systemctl start mysqld

  3. 关闭：systemctl stop mysqld

  4. 重启：systemctl restart mysqld

正则查询

这个比较少用，因为比较耗性能。

普通正则表达式

  • .表示任意一个字符， *表示前面一个字符出现0次 或者多次
  • [abc]表示中括号内任意一个字符，[^abc]表示非 中括号内的字符
  • [0-9]表示数字，[a-z]表示小写字母，[A-Z]表示 大写字母
  • ^xx表示以xx开头，xx$表示以xx结尾
  • [a-zA-Z]表示所有字母，[a-zA-Z0-9]表示所有字 母+数字
  • [^0-9]表示非数字
  • \d表示数字，\D表示非数字
  • \s表示空白字符，\S表示非空白字符

扩展正则表达

在普通正则符号的基础上进行扩展

  • ?表示前面字符出现0或者1次，+前面字符出现1或者多次
  • {a}表示前面字符匹配a次，{a,b}表示前面字符匹配a到b次
  • {,b}表示前面字符匹配0次到b次，{a,}前面字符匹 配a或a+次
  • string1|string2表示匹配string1或者string2

MySQL dump备份数据库

dump备份数据库会锁表，备份前要注意是否影响业务。
数据量大也不建议使用dump,可选择直接复制文件。

准备:我们有test1,test2两个数据库;test1里面有user表

备份

备份多个库

  1. 备份所有库：mysqldump -uroot -pdaokun --all-databases >/tmp/all.sql
  2. 备份多个库：mysqldump -uroot -pdaokun --databases test1 test2 >/tmp/all.sql

备份某个具体库或某个具体表

  1. 备份test1库:mysqldump -uroot -pdaokun test1 >/tmp/test1.sql
  2. 备份test1下的user表:mysqldump -uroot -pdaokun user test1>/tmp/user_test1.sq

特殊备份

  1. 忽略某个表：mysqldump -uroot -pdaokun user --ignore-table=test1.user >/tmp/ignore.sql
  2. 备份表结构：mysqldump -uroot -pdaokun user -d >/tmp/usertruct.sql

MySQL重新初始化

什么情况下需要重新初始化数据库

  1. 学习中创建了很多无用的库，权限分配比较乱
  2. 安装完Mysql，用了一段时间，想废弃之前的数据
  3. 重新初始化会让Mysql恢复到原始的状态

注意:Mysql重新初始化 1. 重新初始化数据会丢失，注意备份 2. 线上环境请勿乱操作

重新初始化步骤

  1. 停止Mysql数据库。/etc/init.d/mysqld stop
  2. 并删除数据目录\rm -rf /data/mysql/*
  3. 重新初始化命令：mysqld --initialize -- user=mysql --basedir=/usr/local/mysql -- datadir=/data/mysql
  4. 记住临时密码：xxx
  5. 启动数据库：/etc/init.d/mysqld restart
  6. 进入Mysql更改密码：alter user 'root'@'localhost' identified with mysql_native_password by 'daokun'
